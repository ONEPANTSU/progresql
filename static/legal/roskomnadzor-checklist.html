2. Перечень обрабатываемых персональных данных

Категории персональных данных, обрабатываемых сервисом ProgreSQL:

Категория данных Конкретные данные Источник получения
Идентификационные Адрес электронной почты (email), имя пользователя Предоставляются субъектом при регистрации
Аутентификационные Хеш пароля (bcrypt, необратимое хеширование) Создаётся при регистрации / смене пароля

Примечание: ProgreSQL не обрабатывает специальные категории персональных данных (расовая, национальная принадлежность, биометрические данные и т.п.) согласно ст. 10 ФЗ-152.

3. Цели обработки персональных данных

  1. Предоставление сервиса — регистрация аккаунта, аутентификация и авторизация пользователей для доступа к функциям ProgreSQL;
  2. Обработка платежей — проведение криптовалютных платежей через CryptoCloud, управление подписками (Free/Pro);
  3. Уведомления — отправка сервисных сообщений: подтверждение email, сброс пароля, уведомления о статусе подписки;
  4. Техническая поддержка — обработка обращений пользователей, устранение неполадок;
  5. Обеспечение безопасности — предотвращение несанкционированного доступа, fraud-защита, журналирование событий;
  6. Учёт использования AI — отслеживание использования токенов LLM-моделей для управления лимитами и биллинга;
  7. Юридическое соответствие — фиксация факта акцепта правовых документов, выполнение требований законодательства РФ.

5. Меры по обеспечению безопасности персональных данных

5.1. Организационные меры

  • Назначение ответственного за организацию обработки ПДн;
  • Утверждение Политики конфиденциальности (опубликована на progresql.com/legal/privacy.html);
  • Получение согласия субъектов при регистрации с записью в таблицу legal_acceptances (дата, IP, User-Agent, версия документа);
  • Определение сроков хранения данных и порядка их уничтожения;
  • Ограничение круга лиц, имеющих доступ к персональным данным.

5.2. Технические меры

  • Хеширование паролей — bcrypt с DefaultCost (необратимое хеширование, пароли в открытом виде не хранятся);
  • Шифрование транспорта — HTTPS/TLS для всех соединений между клиентом и сервером;
  • JWT-аутентификация — токены с подписью HS256, TTL 24 часа, без хранения пароля в токене;
  • Rate limiting — ограничение частоты запросов к API для защиты от перебора;
  • Верификация вебхуков — проверка секретного токена при обработке платёжных уведомлений от CryptoCloud;
  • Constant-time compare — защита от timing-атак при валидации API-ключей;
  • Контроль доступа — серверная инфраструктура с ограниченным доступом;
  • Шифрование паролей БД — локальные пароли подключений к БД шифруются через safeStorage Electron;
  • Автоматическое разъединение — health-check с экспоненциальным backoff и автоотключение при потере связи с БД.

OpenRouter Тексты запросов пользователя (промпты), метаданные схемы БД (имена таблиц, типы колонок) Выполнение AI-запросов (генерация SQL, анализ схемы) США

8. Инструкция по подаче уведомления

Шаг 1. Подготовка

  • Заполните все поля раздела 1 (сведения об операторе) реальными данными;
  • Убедитесь, что Политика конфиденциальности опубликована и доступна по адресу progresql.com/legal/privacy.html;
  • Проверьте, что механизм получения согласия работает (чекбокс при регистрации + запись в legal_acceptances).

Шаг 2. Заполнение формы на портале Роскомнадзора

  1. Перейдите на портал pd.rkn.gov.ru;
  2. Выберите «Подать уведомление об обработке персональных данных»;
  3. Заполните форму, используя данные из разделов 1–7 этого чек-листа;
  4. В поле «Описание мер по обеспечению безопасности» укажите меры из раздела 5;
  5. В поле «Правовое основание» укажите нормы из раздела 4;
  6. Укажите дату начала обработки — дату запуска сервиса;
  7. В разделе трансграничной передачи укажите OpenRouter (США) и CryptoCloud.

Шаг 3. Подписание и отправка

  • Подпишите уведомление усиленной квалифицированной электронной подписью (УКЭП) или направьте бумажный экземпляр почтой;
  • Адрес для бумажного направления: 109992, г. Москва, Китайгородский проезд, д. 7, стр. 2, Роскомнадзор;
  • Срок рассмотрения — до 30 дней с момента получения.

Шаг 4. После подачи

  • Сохраните подтверждение подачи (номер уведомления, дату);
  • При изменении перечня ПДн, целей обработки или мер защиты — подайте информационное письмо об изменениях;
  • При прекращении обработки ПДн — подайте уведомление о прекращении.
Статья 22 ФЗ-152: оператор обязан уведомить Роскомнадзор до начала обработки персональных данных, за исключением случаев, предусмотренных ч. 2 ст. 22. Рекомендуется подать уведомление в любом случае для минимизации рисков.

9. Архитектура ProgreSQL (для понимания потоков данных)

Краткое описание архитектуры для корректного заполнения уведомления:

  • Клиент: десктопное приложение (Electron + React), устанавливается пользователем локально;
  • Backend: Go-сервер (REST API + WebSocket), размещён на сервере оператора;
  • База данных: PostgreSQL — хранит таблицы users, payments, token_usage, legal_acceptances;
  • Аутентификация: email + пароль → JWT-токен (HS256, TTL 24ч);
  • Платежи: CryptoCloud API — криптовалютные платежи, вебхуки для подтверждения;
  • AI: OpenRouter API — передача пользовательских запросов и метаданных схемы БД для генерации SQL;
  • Email: Yandex SMTP (smtp.yandex.ru:465) — отправка сервисных писем;
  • Данные пользовательских БД: обрабатываются только локально в десктопном приложении, не передаются на сервер оператора (кроме метаданных схемы для AI-запросов).
